    Arrival Detection
    Instead of stopping at a fixed pixel distance, the bot captures a region around the player and checks if the target image appears inside it — much more accurate for game objects that vary in size or position.
    Click Load Screen to capture your screen and position the arrival region
    Drag corners to resize · Drag center to move
    Arrival region width (px)
    Arrival region height (px)
    Arrival confidence
    Arrival region width / height — size of the scan box around the player. Set width and height independently for a rectangular zone. Larger = stops earlier.
    Arrival confidence — how sure the match must be inside that box to stop. Higher = must be very close.
    Movement Parameters
    • Step time – how long each key hold lasts (e.g., 0.1s). Smaller = smoother but slower overall.
    • Stop radius – pixel-distance fallback if arrival region check fails.
    • Stuck threshold – number of iterations without progress before attempting a small opposite move to break free.

    Script Writing Guide — Zero to Hero
    A macro script is a text file where each line is one instruction. The app reads it top to bottom and does each action in order. Lines starting with # are comments — the app ignores them.
    Golden Rules
    • One command per line
    • Commands are ALL CAPS (e.g. CLICK, WAIT)
    • Arguments come after the command separated by a space
    • Use # to write notes to yourself
    • Indentation inside blocks is optional but recommended for readability
    CommandWhat it doesExample
    WAIT_RANDOMWait a random duration between min and max seconds (anti-detect)WAIT_RANDOM 1.0 3.0
    WAITPause for N seconds before next actionWAIT 2
    WAIT 0.5You can use decimals for millisecondsWAIT 0.5
    # commentA note — app skips this line completely# click the button
    Tip: Always start your script with WAIT 2 to give yourself time to switch to the target window after pressing Run.
    # My first macro WAIT 2 # wait 2 seconds before starting CLICK 500 300 # click at x=500, y=300 TYPE Hello # type the word Hello PRESS enter # press the Enter key
    All mouse commands need X and Y coordinates on your screen. X = distance from left edge. Y = distance from top edge. Use the Windows Snipping Tool or hover your mouse in any image editor to find coordinates.
    CommandArgumentsExample
    CLICK x y CLICK 500 300
    DOUBLE_CLICK x y DOUBLE_CLICK 500 300
    RIGHT_CLICK x y RIGHT_CLICK 500 300
    MOVE x y MOVE 800 400
    MOVE_HUMAN x y MOVE_HUMAN 800 400 — curved path
    SCROLL amount (+ = up, - = down) SCROLL -3
    DRAG x1 y1 x2 y2 DRAG 100 200 500 200
    How to find X Y coordinates:
    Open Paint → hover your mouse → the bottom bar shows the coordinates. Or use the Quick Detect panel — it shows the detected position.
    # Mouse example MOVE 500 300 # move mouse (no click) WAIT 0.5 CLICK 500 300 # left click DOUBLE_CLICK 500 300 # double click same spot SCROLL -5 # scroll down 5 clicks DRAG 100 200 400 200 # drag left to right

    Image commands find a saved template image on your screen and interact with it. You must first capture a template using the Draw Region on Screen button.
    Step-by-step: Use image detection
    1. Go to the Image Capture section on the left
    2. Type a name like start_button
    3. Click Draw Region on Screen
    4. Drag a rectangle around the button or image on your screen
    5. The template is saved — now use it in your script
    CommandArgumentsWhat it does
    CLICK_IMAGE template_name [confidence] Find image and left click it
    DOUBLE_CLICK_IMAGE template_name Find image and double click it
    RIGHT_CLICK_IMAGE template_name Find image and right click it
    CLICK_RANDOM_OFFSET template_name [radius=8] [confidence] Find image and click within a random radius (anti-detect)
    DOUBLE_CLICK_RANDOM_OFFSET template_name [radius=8] [confidence] Find image and double-click within a random radius
    RIGHT_CLICK_RANDOM_OFFSET template_name [radius=8] [confidence] Find image and right-click within a random radius
    WAIT_IMAGE template_name [timeout_seconds] Pause until image appears on screen
    WAIT_IMAGE_GONE template_name [timeout_seconds] Pause until image disappears
    Confidence is how similar the match must be (0.0–1.0). Default is 0.8 (80%). If detection fails, try lowering it to 0.6. If it triggers on wrong things, raise it to 0.9.
    # Image detection example WAIT_IMAGE start_button 30 # wait up to 30s for it CLICK_IMAGE start_button # click it when found WAIT_IMAGE_GONE loading 60 # wait for loading to finish CLICK_IMAGE play_button 0.7 # lower confidence = 70%
    NAVIGATE_TO_IMAGE moves your game character toward a target object on screen using keyboard movement keys. It continuously detects the target and adjusts direction until it arrives.
    Setup — do this first
    1. Go to Movement AI tab
    2. Set Player Position — the pixel on screen that is always your character (e.g. crosshair center, usually 960×540 on 1080p)
    3. Set Movement Keys — which keys move your character (WASD or arrow keys)
    4. Set Target Window — click Use active window while your game is focused, then Save Settings
    5. Capture the template image of the object you want to navigate to (enemy, item, door…)
    Full command syntax
    NAVIGATE_TO_IMAGE template # simplest form — uses all defaults NAVIGATE_TO_IMAGE ore confidence=0.8 NAVIGATE_TO_IMAGE ore confidence=0.8 arrival_region=150 NAVIGATE_TO_IMAGE ore confidence=0.8 miss_tolerance=10 timeout=30
    ParameterDefaultWhat it does
    confidence0.8How similar the template must look to trigger (0.0–1.0). Lower = easier to match but more false positives.
    arrival_region200Width in pixels of the box around the player to scan. When the target appears inside this box, navigation stops.
    arrival_region_harrival_regionHeight in pixels of the arrival box. If omitted, defaults to arrival_region (square). Set independently for a rectangular zone.
    arrival_confidence0.85Confidence required inside the arrival region to count as arrived. Higher = must be very close.
    miss_tolerance3How many consecutive frames the target can be missing before navigation stops. Increase for games with loading screens or brief UI overlays.
    timeout0Maximum seconds before giving up. 0 = no timeout.
    offsetX0Move toward a point this many pixels right of the template center.
    offsetY0Move toward a point this many pixels below the template center. Use negative to target above it.
    Tip: Use the Navigate tab in the Image Action builder — it has sliders for all parameters and shows a live command preview.
    🎮 Common game automation patterns
    Navigate to item, then pick it up:
    NAVIGATE_TO_IMAGE ore confidence=0.75 miss_tolerance=8 WAIT 0.3 PRESS f # interact key
    Loop — keep navigating to nearest enemy:
    LOOP NAVIGATE_TO_IMAGE enemy confidence=0.8 timeout=10 miss_tolerance=6 PRESS space # attack WAIT 0.5 END
    Navigate with offset — stop 20px above target:
    NAVIGATE_TO_IMAGE chest 0.8 offsetY=-20
    Auto window focus: The app automatically focuses your game window when the script starts — you don't need to click the game first. Configure the target window in the Movement AI tab.
    Logic commands let your script make decisions and repeat actions. Every block that opens must be closed with END.
    IF / ELSE — Make a decision
    If the image is on screen → do one thing. If not → do another.
    IF_IMAGE enemy.png PRESS space # runs if enemy.png IS found WAIT 0.2 ELSE WAIT 1 # runs if enemy.png is NOT found END
    Check the opposite — run if image is not visible:
    IF_NOT_IMAGE door_open.png CLICK_IMAGE open_door_btn END
    REPEAT — Do something N times
    REPEAT 5 # repeat the block 5 times CLICK 300 400 WAIT 0.5 END
    ♾️ LOOP — Repeat forever until you press Stop
    LOOP IF_IMAGE enemy.png PRESS space ELSE WAIT 0.5 END END
    WHILE_IMAGE — Loop while image is visible
    WHILE_IMAGE loading_bar WAIT 1 # keep waiting while bar is visible END CLICK_IMAGE start_button # runs after loading finishes
    Remember: every IF, LOOP, REPEAT, and WHILE must end with END. You can nest them inside each other.
    Variables let you store values and reuse them throughout your script. They make your macros dynamic and adaptable.
    Syntax
    CommandDescriptionExample
    SET var = value Assign a value to a variable. The value can be a number, a string, or an expression. SET x = 500
    $var Use the variable anywhere in a command. It will be replaced by its current value. CLICK $x 300
    Expressions

    You can use simple arithmetic in SET expressions: + - * /. The expression is evaluated when the line runs.

    SET x = 5 + 3 * 2 SET y = $x / 2 CLICK $x $y
    Limitation: Expressions do not support parentheses or multiple operators in complex orders – they are evaluated strictly left‑to‑right. For example, 5 + 3 * 2 gives 16, not 11. Use separate SET steps for complex calculations.
    Strings

    You can store text in variables and use them where a text argument is expected (e.g., TYPE). Strings can contain spaces if you enclose them in quotes.

    SET name = "John Doe" TYPE Hello $name
    Using Variables in Image Commands

    Variables can be used for template names, confidence values, offsets, etc.

    SET button = start_button SET conf = 0.85 FIND_CLICK $button confidence=$conf offsetX=10
    Dynamic Click Example
    SET x = 500 SET y = 300 REPEAT 10 CLICK $x $y SET x = $x + 10 WAIT 0.2 END
    IF_VAR — Branch on a variable value

    Compare a variable against a value and run different blocks depending on the result.

    IF_VAR $score >= 100 TOAST You win! info ELSE TOAST Keep going... warn END

    Supported operators: == != < > <= >= contains startswith endswith

    🔁 WHILE_VAR — Loop while a condition is true
    SET count = 0 WHILE_VAR $count < 5 CLICK 500 300 SET count = $count + 1 WAIT 0.5 END
    ⏭ REPEAT_UNTIL — Keep trying until a condition is met

    Runs the body at least once, then checks the condition. Three variants:

    # Until an image appears REPEAT_UNTIL IMAGE login_success CLICK 400 300 WAIT 2 END # Until a color appears REPEAT_UNTIL COLOR #27ae60 WAIT 1 END # Until a variable condition is true REPEAT_UNTIL VAR $hp_found == 1 READ_TEXT "Health:" -> $hp WAIT 1 END
    ⚠️ ON_ERROR — Handle failures gracefully

    Place an ON_ERROR … END block immediately after any command that might fail. The block runs only if the previous command threw an error or failed to find its target.

    FIND_CLICK login_button ON_ERROR TOAST Login button not found — retrying warn WAIT 3 FIND_CLICK login_button END
    OCR commands find text on screen using Tesseract and perform actions at its location. No template images needed — just type the text you want to find.
    Text Action Commands
    CommandWhat it does
    TEXT_CLICK "text"Find text on screen and left-click it
    TEXT_DOUBLE_CLICK "text"Find text and double-click it
    TEXT_RIGHT_CLICK "text"Find text and right-click it
    TEXT_MOVE "text"Move mouse to text (no click)
    TEXT_DRAG "text" x yDrag from text to destination x y

    All TEXT_ commands support optional parameters:

    TEXT_CLICK "Start Game" confidence=70 region=0 0 800 600 TEXT_CLICK "OK" anchor=top_left offsetX=5 offsetY=-2
    READ_TEXT — Store text location in a variable

    Find text on screen and save its coordinates and a found-flag into variables you can use later.

    READ_TEXT "Health:" -> $hp # Now you can use: # $hp = "Health:" (the matched text) # $hp_found = 1 (found) or 0 (not found) # $hp_x, $hp_y, $hp_w, $hp_h = bounding box IF_VAR $hp_found == 1 CLICK $hp_x $hp_y END
    Retry until a button appears
    REPEAT_UNTIL VAR $found == 1 READ_TEXT "Confirm" -> $found WAIT 1 END TEXT_CLICK "Confirm"
    Tip: Lower the confidence value (e.g. confidence=60) if the text is on a busy background or in a non-standard font. Use the Text (OCR) accordion in the Macro Editor to scan and test before scripting.
    Color commands find a pixel of a specific color on screen and act on it. Great for detecting UI states (health bars, indicators, status lights) without needing template images.
    Color Action Commands
    CommandWhat it does
    COLOR_CLICK #hexFind first pixel of that color and click it
    COLOR_MOVE #hexMove mouse to the color pixel (no click)
    COLOR_RIGHT_CLICK #hexRight-click the color pixel
    COLOR_DRAG #hex x yDrag from color pixel to destination
    WAIT_COLOR #hexWait until that color appears on screen
    READ_COLOR x y -> $varRead the pixel color at x,y into a variable

    Optional parameters:

    COLOR_CLICK #27ae60 tolerance=30 COLOR_CLICK #FF0000 tolerance=20 region=0 0 960 540 WAIT_COLOR #00FF00 tolerance=15 timeout=10
    READ_COLOR — Sample a pixel into a variable
    READ_COLOR 500 300 -> $px # $px = "#FF0000" (hex string) # $px_r = 255 (red channel) # $px_g = 0 (green channel) # $px_b = 0 (blue channel)
    Wait for green light, then click
    # Keep waiting until a green indicator appears WAIT_COLOR #27ae60 tolerance=30 timeout=30 COLOR_CLICK #27ae60 tolerance=30 TOAST Clicked the green button! info
    Tip: Use the Color Pixel tab in the Macro Editor to pick a color from the screen with the eyedropper, then scan to verify it works before scripting. Start with tolerance=30 and lower it if too many false matches appear.
    Clipboard commands let your macros copy and paste text programmatically — reading values from screen via OCR and moving them to other applications.
    📋 Clipboard Commands
    CommandWhat it does
    CLIPBOARD_SET "text"Put text into the clipboard. Supports $variables.
    CLIPBOARD_GET -> $varRead the current clipboard content into a variable
    CLIPBOARD_COPYSend Ctrl+C (copies current selection). Result stored in $_clipboard
    CLIPBOARD_PASTESend Ctrl+V (pastes clipboard)
    Requires pyperclip

    Install it once with:

    pip install pyperclip
    Copy a value and paste it elsewhere
    # Click a field, select all, copy it CLICK 400 300 HOTKEY ctrl+a CLIPBOARD_COPY CLIPBOARD_GET -> $value # Click the destination field and paste CLICK 400 500 CLIPBOARD_SET $value CLIPBOARD_PASTE
    Read text via OCR, paste into another field
    # Read a player name from the screen READ_TEXT "Player:" -> $label # Copy whatever comes after "Player:" label CLIPBOARD_SET $label CLICK 600 400 CLIPBOARD_PASTE
    Several tools help you find and fix problems in your scripts without running the whole thing each time.
    Run from Cursor — fastest debugging tool

    Click anywhere in your script, then click Run from Cursor next to the Run button. The script starts from that exact line — no waiting through the whole script. Also available by right-clicking anywhere in the editor.

    Breakpoint debug mode
    1. Set breakpoints — click any line number in the line panel. A red dot ● appears.
    2. Start debugging — click the 🐞 Debug button.
    3. While paused: ⏯ Step runs the current line, ▶ Continue runs to next breakpoint, 📋 Variables opens the watch panel where every variable updates in real time.
    No breakpoints? Debug acts as step-through mode — pauses before every line.
    Log panel — filter the noise
    • ⚠ Warn filter — shows missed detections, navigation misses only
    • ✕ Error filter — shows only crashes and hard errors
    • Search box — type a template name like enemy to see only lines about it
    🔍 Live Detect — see what the bot sees

    Go to Templates → select a template → Step 3 → click Live Detect. Shows colour-coded boxes in real time: green = strong, yellow = medium, red = weak. Use the fps slider for smoother preview on a fast PC.

    Debug a variable-driven loop
    SET count = 0 WHILE_VAR $count < 3 # set breakpoint here CLICK 500 300 SET count = $count + 1 WAIT 0.5 END
